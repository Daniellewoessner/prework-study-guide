HTML

  • The head element contains information about webpage
  • The body element represents the visible content shown to the user
  • HTML elements work together to build the structure of a webpage and provide framework that makes it easy for a user to navigate the webpage
  • HTML attributes are always defined in the beggining elements, never the closing element
  • Declarations are not case senstitive but elements are case sensitive.
  • Use semantic HTML elements when possible to make HTML easier to read
  • Elements are nested inside other elements are called "child elements." Elements that hold children are called "parent elements"

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Increasing padding makes the entire element bigger.
  • Asterrisk * is a special symbol that declares ALL elements on the page will have style properties
  • Three Ways to style a webpage
      1. Inline CSS: Style element directly into HTML file. Useful for quick and premanenet changes. Less flexible than external/internal style sheets. Each inline style must be edited individually whenever you make a design change.
      2. Internal Style Sheet: Embeded the entire stylesheet into HTML file using style element in the head element.
      3. External Style Sheet: Most common; allow developers to keep all their CSS rules in a seperate file. Design changes are easier. Link the file using the link element placed into the HTMl head.
  • DRY= Do Not Repeat Yourself; reduce overall size and complexity of code when possible
  • Class attribute; add style to certain elements but not all elements. Can be assigned to the element in the HTML (ex.card)
Git
  • Git status: Check what branches we are currently on
  • Git checkout-b branch name: creates a new branch and switches to it
  • git checkout main: pull down main branch
  • Workflow for Content Updates
      • 1.Creates a branch
      2.Open a pull request.
      3. Merge and deploy.
  • Git commit: take all of the changes that have been made locally and push them up to a remote repository.
Javascript
  • Allows users to interact with website.
  • Browser reads code in the order it appears in the file.
  • Case sensitive
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • /* Comments*/ // This is a comment
  • Strict equality ===
  • Conditionals are if/else statements.
  • Functions are set of instructions that are packaged and reused. Must call functions by name to use it
  • Undefined (absense of value; indentifier); Null (absence of an object(keyword)
  • Link script.js to html in 'body'.
  • Chrome Dev TOols: CTRL+SHIFT=1 (inspect)
  • Arrays are single variables that are used to hold a group of data and can hold different types of data.
